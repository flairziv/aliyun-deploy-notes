# 9. 日常更新与维护

## 更新流程速查

### 后端代码改了

```cmd
:: 1. Windows 本地打包
cd ai-hardware-server
mvn clean package -DskipTests

:: 2. WinSCP 上传
:: target\ai-hardware-server-0.0.1-SNAPSHOT.jar → /opt/ai-hardware/app.jar
```

```bash
# 3. 服务器上重启
systemctl restart ai-hardware

# 4. 确认启动成功
systemctl status ai-hardware
```

### 前端代码改了

```cmd
:: 1. Windows 本地打包
cd ai-hardware-ui
npm run build

:: 2. WinSCP 上传
:: dist\ 下所有文件 → /var/www/ai-hardware/（覆盖）
```

> 前端更新**不需要重启任何服务**，浏览器刷新即可。有缓存按 `Ctrl+Shift+R`。

### 数据库结构改了

```bash
# 上传新的 schema.sql 后执行
mysql -u ai_hw -p ai_hardware < /opt/ai-hardware/schema.sql
```

> **警告**：`schema.sql` 包含 `DROP TABLE`，重新导入会清空所有数据！如果只改某条数据，建议直接用 SQL 修改。

### 修改数据库密码

```bash
# 1. MySQL 里改密码
mysql -u root -p
ALTER USER 'ai_hw'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
EXIT;

# 2. 修改配置文件
nano /etc/ai-hardware/application-prod.yml
# 更新 password 字段

# 3. 重启后端
systemctl restart ai-hardware
```

## 查看日志

```bash
# 实时查看后端日志
journalctl -u ai-hardware -f

# 查看最近 100 行日志
journalctl -u ai-hardware -n 100

# 查看 Nginx 访问日志
tail -f /var/log/nginx/access.log

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log
```

## 部署检查清单

每次部署完成后逐项验证：

- [ ] 访问首页正常加载
- [ ] 页面刷新不出现 404
- [ ] 数据正常显示（API 接口通畅）
- [ ] 图片正常显示
- [ ] 后台管理登录正常
- [ ] 文件上传功能正常
- [ ] `systemctl status ai-hardware` 显示 active (running)
- [ ] 服务器重启后后端自动启动
