# 10. 踩坑记录与排错

## 问题 1：YAML 密码含 @ 导致启动失败

**报错**：

```
found character '@' that cannot start any token. (Do not use @ for indentation)
```

**原因**：YAML 中 `@` 是特殊字符，密码 `密码` 没有被引号包裹。

**解决**：用单引号包裹：

```yaml
password: '密码'
```

---

## 问题 2：SQL 导入报语法错误

**报错**：

```
ERROR 1064 (42000) at line 413
```

**原因**：INSERT 语句中，某条数据被注释掉了，但上一条数据末尾的逗号没有改成分号。

**修复前**：

```sql
('文章2', ...),     -- 逗号，期待下一条
-- ('文章3', ...);  -- 被注释了
```

**修复后**：

```sql
('文章2', ...);     -- 改成分号，结束 INSERT
-- ('文章3', ...);
```

## 问题 3：访问域名显示 Nginx 默认页面

**原因**：`/etc/nginx/sites-enabled/default` 抢先匹配了请求。

**解决**：

```bash
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx
```

---

## 问题 4：HTTPS 配置后 443 端口无响应

**原因**：certbot 把 HTTPS 配置写进了 `default` 文件，删除 `default` 后 443 端口没有配置了。

**解决**：将 HTTP 跳转和 HTTPS 配置合并写入 `ai-hardware` 配置文件中（参考 [Nginx 配置详解](07-nginx.md) 和 [HTTPS 配置](08-https-domain.md)）。

## 问题 5：后端日志暴露 SQL 和用户数据

**表现**：

```
==>  Preparing: SELECT id, username, password ...
==> Parameters: admin(String)
<==    Row: 1, admin, $2a$10$xxxx ...
```

**原因**：MyBatis 的 `StdOutImpl` 日志在生产环境没有关闭。

**解决**：在 `application.yml` 中使用条件配置：

```yaml
mybatis:
  configuration:
    log-impl: ${MYBATIS_LOG:org.apache.ibatis.logging.nologging.NoLoggingImpl}
```

- 生产环境默认不打印 SQL
- 本地调试时设置环境变量 `MYBATIS_LOG=org.apache.ibatis.logging.stdout.StdOutImpl` 开启

## 问题 6：`conflicting server name` 警告

**原因**：多个配置文件定义了相同的 `server_name`。

**排查**：

```bash
grep -rl "你的域名" /etc/nginx/sites-available/ /etc/nginx/sites-enabled/
```

**解决**：确保只有一个文件包含你的域名，删除或注释掉其他文件中的重复配置。

---

## 问题 7：页面加载慢

**原因**：Nginx 没有开启 gzip 压缩，JS/CSS 等文件以原始大小传输。

**解决**：编辑 `/etc/nginx/nginx.conf`，开启 gzip：

```nginx
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_types text/plain text/css application/json application/javascript
           text/xml application/xml text/javascript image/svg+xml;
```

```bash
nginx -t && systemctl reload nginx
```

## 问题 8：certbot 提示证书未到期

**提示**：`Certificate not yet due for renewal`

**原因**：证书还没到期，不需要续期。选择 `1: Attempt to reinstall this existing certificate` 把现有证书重新绑定到 Nginx 配置即可。
