# 4. 数据库初始化

## 创建数据库和用户

先将本地的 `schema.sql` 上传到服务器 `/opt/ai-hardware/schema.sql`，然后：

```bash
# 以 root 身份进入 MySQL
mysql -u root -p
```

执行以下 SQL：

```sql
-- 创建数据库
CREATE DATABASE ai_hardware CHARACTER SET utf8mb4;

-- 创建专用用户（不要用 root 连接业务数据库）
CREATE USER 'ai_hw'@'localhost' IDENTIFIED BY '你的强密码';

-- 授权
GRANT ALL ON ai_hardware.* TO 'ai_hw'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

## 为什么要创建专用用户？

- `root` 用户权限太大，如果应用被攻破，攻击者可以操作所有数据库
- `ai_hw` 用户只能操作 `ai_hardware` 这一个数据库，即使泄露也不会波及其他数据
- `@'localhost'` 表示只允许本机连接，外部无法远程登录

## 导入表结构和初始数据

```bash
mysql -u ai_hw -p ai_hardware < /opt/ai-hardware/schema.sql
```

- `-u ai_hw`：用 ai_hw 用户登录
- `-p`：提示输入密码
- `ai_hardware`：指定要导入的数据库
- `< schema.sql`：将 SQL 文件内容导入执行

## 验证导入是否成功

```bash
mysql -u ai_hw -p ai_hardware -e "SHOW TABLES;"
```

应该能看到所有业务表列表。

## 踩坑：SQL 语法报错

**报错信息**：

```
ERROR 1064 (42000) at line 413: You have an error in your SQL syntax
```

**原因**：INSERT 语句的最后一条数据以逗号 `,` 结尾，而紧接着的下一条数据被注释掉了，导致整个 INSERT 没有分号 `;` 结束。

**错误示例**：

```sql
INSERT INTO `news` VALUES
('文章1', ...),
('文章2', ...),    -- ← 这里的逗号是问题所在
-- ('文章3', ...);  -- 这条被注释了，但上面的逗号还在
```

**修复**：把最后一条有效数据末尾的逗号改成分号：

```sql
('文章2', ...);    -- ← 改成分号
```

> **经验**：注释掉 INSERT 中的某些行时，一定要检查前一行的逗号是否需要改成分号。
