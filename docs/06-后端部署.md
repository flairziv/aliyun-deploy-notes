# 5. 后端部署

## 创建生产配置文件

生产环境的数据库密码、JWT 密钥等敏感信息不应该写在代码里，而是放在服务器上的独立配置文件中。

```bash
nano /etc/ai-hardware/application-prod.yml
```

核心配置项：

```yaml
server:
  port: 48080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ai_hardware?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: ai_hw
    password: '你的数据库密码'    # 有特殊字符要加引号
    driver-class-name: com.mysql.cj.jdbc.Driver

jwt:
  secret: 用openssl生成的随机字符串
  expire-seconds: 86400

notify:
  enabled: false    # 没配置邮箱就设为 false

logging:
  level:
    com.aiot.aihardwareserver: info
```

设置文件权限，只有 root 可读写：

```bash
chmod 700 /etc/ai-hardware
chmod 600 /etc/ai-hardware/application-prod.yml
```

## 踩坑：YAML 中的特殊字符

**报错**：

```
found character '@' that cannot start any token
```

**原因**：密码中包含 `@` 等特殊字符（`@`、`#`、`:`、`{`、`}` 等），YAML 会将其解析为特殊语法。

**解决**：用单引号包裹密码值：

```yaml
password: '密码'    # 单引号内的内容不会被 YAML 解析
```

## 生成 JWT 密钥

```bash
openssl rand -hex 32
```

输出一串 64 位的随机十六进制字符串，复制到配置文件的 `jwt.secret` 中。

> JWT 密钥用于签发和验证登录 token，一旦更换，所有在线用户需要重新登录。

## 测试启动

```bash
java -jar /opt/ai-hardware/app.jar \
  --spring.config.additional-location=/etc/ai-hardware/application-prod.yml
```

看到 `Started ... in X seconds` 表示启动成功。`Ctrl+C` 停止。

## 配置 systemd 开机自启

创建服务文件：

```bash
nano /etc/systemd/system/ai-hardware.service
```

```ini
[Unit]
Description=AI Hardware Backend
After=network.target mysql.service

[Service]
User=root
WorkingDirectory=/opt/ai-hardware
ExecStart=/usr/bin/java -jar /opt/ai-hardware/app.jar \
  --spring.config.additional-location=/etc/ai-hardware/application-prod.yml
Restart=on-failure
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
systemctl daemon-reload
systemctl enable ai-hardware
systemctl start ai-hardware
```

常用操作：

```bash
systemctl status ai-hardware      # 查看状态
systemctl restart ai-hardware     # 重启
journalctl -u ai-hardware -f      # 实时日志
```

## 配置文件分离的好处

| | 代码中的 application.yml | 服务器上的 application-prod.yml |
|---|---|---|
| 数据库密码 | 占位符（如 `${DB_PASSWORD:dev_pwd}`） | 真实的生产密码 |
| JWT 密钥 | 开发用默认值 | openssl 生成的随机密钥 |
| 谁能看到 | 任何拿到代码的人（但只有占位符） | 只有服务器 root（权限 600） |

**核心原则**：代码仓库里绝不包含生产密码，即使代码泄露也不影响线上安全。
