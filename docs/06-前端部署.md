# 6. 前端部署

## 上传文件

前端打包后是纯静态文件，不需要运行任何进程，只需要让 Nginx 指向这些文件就行。

```bash
# 在服务器上创建目录
mkdir -p /var/www/ai-hardware
```

用 WinSCP 上传：

| 本地路径 | 服务器路径 |
|---------|-----------|
| `ai-hardware-ui\dist\` 下所有文件 | `/var/www/ai-hardware/` |
| `ai-hardware-ui\public\images\` 下所有图片 | `/var/www/ai-hardware/images/` |

## 上传后目录结构

```
/var/www/ai-hardware/
├── index.html
├── assets/
│   ├── index-xxxx.js
│   ├── index-xxxx.css
│   └── ...
└── images/
    ├── banner1.png
    ├── banner2.png
    └── ...
```

## 为什么前端更新不需要重启服务？

因为前端是纯静态文件，Nginx 直接读取磁盘上的文件返回给浏览器。覆盖文件后，下次请求就会拿到新内容。

如果浏览器有缓存，按 `Ctrl+Shift+R` 强制刷新即可。

## 与后端部署的区别

| | 前端 | 后端 |
|---|---|---|
| 本质 | HTML/JS/CSS 静态文件 | Java 进程 |
| 运行方式 | Nginx 直接返回文件 | 需要 JVM 运行 jar 包 |
| 更新方式 | 覆盖文件即可 | 上传 jar 后需重启服务 |
| 需要重启？ | 不需要 | 需要 `systemctl restart` |
