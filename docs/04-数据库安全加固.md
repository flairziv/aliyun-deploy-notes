# 04 - 数据库安全加固

## 为什么需要这一步

`apt install mysql-server` 安装完 MySQL 后，默认存在匿名用户、test 数据库等不安全配置。虽然 Ubuntu 下 MySQL root 默认使用 `auth_socket` 认证（只有系统 root 用户才能免密登录），但仍需要通过官方安全脚本清理这些隐患。

## 执行安全初始化

```bash
mysql_secure_installation
```

按提示依次操作：

| 提示 | 选择 | 说明 |
|------|------|------|
| Setup VALIDATE PASSWORD component? | Y | 开启密码强度校验 |
| Password validation policy (0/1/2) | 2 (STRONG) | 要求长度>=8、含大小写、数字、特殊字符 |
| Remove anonymous users? | Y | 删除匿名用户，防止未授权访问 |
| Disallow root login remotely? | Y | 禁止 root 远程登录 |
| Remove test database? | Y | 删除测试数据库 |
| Reload privilege tables? | Y | 刷新权限使配置立即生效 |

## 关于 root 密码跳过的说明

执行过程中可能看到这段提示：

```
Skipping password set for root as authentication with auth_socket is used by default.
```

这是正常的。`auth_socket` 的含义：

- MySQL root 账户绑定到 Linux 系统的 root 用户
- 只有通过 `sudo mysql` 或以 root 身份执行 `mysql` 才能登录
- 外部网络、其他系统用户均无法连接 MySQL root
- 不需要额外设置 root 密码，安全性已有保障

## 验证

执行完成后可以验证：

```bash
# 以 root 身份登录 MySQL（无需密码）
mysql -u root -p
# 直接回车即可进入，说明 auth_socket 生效

# 检查匿名用户是否已删除
SELECT user, host FROM mysql.user;
# 应该只看到 root 和你创建的业务用户，没有空用户名的行
```

## 学习笔记

- `mysql_secure_installation` 是 MySQL 官方提供的安全加固脚本，生产环境必须执行
- `auth_socket` 是 Ubuntu/Debian 系统的默认认证方式，比密码认证更安全（无密码可被暴力破解）
- 这一步应该在创建业务数据库和用户**之前**完成
- 业务应用（如 Spring Boot）使用的是单独创建的 `ai_hw` 用户，不受 root 认证方式影响
